# 最右/2: 网格迷宫最短路径问题

## 题目描述

小右是一位充满冒险精神的探险家，进入了一座神秘的迷宫。这个迷宫由一个 n×m 的矩阵组成，其中：
- 0 代表可以通行的道路
- 1 代表无法穿越的障碍物

需要从迷宫入口（左上角 (0, 0)）出发，前往终点（右下角 (n-1, m-1)）。

**约束条件**：
- 每次只能向右或向下移动
- 求从起点到终点最短路径的长度
- 如果无法到达终点，返回 -1

**示例**：
```
迷宫:
[0, 0, 0]
[0, 1, 0]
[0, 0, 0]

输出: 4
解释: 最短路径为 (0,0) → (1,0) → (2,0) → (2,1) → (2,2)，步数为4
```

## 解题思路

### 核心思想
使用动态规划计算每个位置到起点的最短距离。由于只能向右或向下移动，这个问题具有**无后效性**，可以用DP解决。

### 算法步骤

1. **初始化**：
   ```
   dp[i][j] = 到达位置(i,j)所需的步数
   dp[0][0] = 0  // 起点无需移动
   ```

2. **初始化第一列**（只能从上方来）：
   ```
   for i = 1 to n-1:
       if grid[i][0] == 0 and dp[i-1][0] != -1:
           dp[i][0] = dp[i-1][0] + 1
       else:
           dp[i][0] = -1  // 无法到达
   ```

3. **初始化第一行**（只能从左方来）：
   ```
   for j = 1 to m-1:
       if grid[0][j] == 0 and dp[0][j-1] != -1:
           dp[0][j] = dp[0][j-1] + 1
       else:
           dp[0][j] = -1
   ```

4. **填充dp数组**：
   对于位置(i, j)，可以从上方(i-1, j)或左方(i, j-1)来：
   ```
   if grid[i][j] == 1:
       dp[i][j] = -1  // 障碍物无法到达
   else:
       up = dp[i-1][j]
       left = dp[i][j-1]
       
       if both are -1:
           dp[i][j] = -1
       elif up == -1:
           dp[i][j] = left + 1
       elif left == -1:
           dp[i][j] = up + 1
       else:
           dp[i][j] = min(up, left) + 1
   ```

5. **返回结果**：`dp[n-1][m-1]`

### 时间复杂度
- **时间**: $O(n \times m)$ - 遍历每个网格位置
- **空间**: $O(n \times m)$ - DP数组大小

## 代码实现

```go
func shortestPath(grid [][]int) int {
    if len(grid) == 0 || len(grid[0]) == 0 {
        return -1
    }
    
    n, m := len(grid), len(grid[0])
    
    // 起点或终点是障碍物
    if grid[0][0] == 1 || grid[n-1][m-1] == 1 {
        return -1
    }
    
    // 初始化dp数组
    dp := make([][]int, n)
    for i := range dp {
        dp[i] = make([]int, m)
    }
    dp[0][0] = 0
    
    // 初始化第一列
    for i := 1; i < n; i++ {
        if grid[i][0] == 0 && dp[i-1][0] != -1 {
            dp[i][0] = dp[i-1][0] + 1
        } else {
            dp[i][0] = -1
        }
    }
    
    // 初始化第一行
    for j := 1; j < m; j++ {
        if grid[0][j] == 0 && dp[0][j-1] != -1 {
            dp[0][j] = dp[0][j-1] + 1
        } else {
            dp[0][j] = -1
        }
    }
    
    // 填充dp数组
    for i := 1; i < n; i++ {
        for j := 1; j < m; j++ {
            if grid[i][j] == 1 {
                dp[i][j] = -1
                continue
            }
            
            up, left := dp[i-1][j], dp[i][j-1]
            
            if up == -1 && left == -1 {
                dp[i][j] = -1
            } else if up == -1 {
                dp[i][j] = left + 1
            } else if left == -1 {
                dp[i][j] = up + 1
            } else {
                if up < left {
                    dp[i][j] = up + 1
                } else {
                    dp[i][j] = left + 1
                }
            }
        }
    }
    
    return dp[n-1][m-1]
}
```

## 重要细节

### -1 的含义
-1 表示该位置无法到达。需要特殊处理以避免覆盖有效的路径。

### 状态合并
计算 `dp[i][j]` 时，取从上方和左方来的最小值：
```
dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + 1
```

### 边界处理
- 起点或终点是障碍物时直接返回 -1
- 第一行和第一列需要单独处理（只有一个来向）

## 算法对比

### BFS（广度优先搜索）
- 适用于无权图的最短路径
- 时间：$O(nm)$，空间：$O(nm)$
- 代码相对复杂

### DP（动态规划）
- 利用了只能向右下的约束
- 时间：$O(nm)$，空间：$O(nm)$
- 代码更简洁直观

### 空间优化
由于每个位置只依赖上方和左方，可以优化为 $O(m)$：
```go
dp := make([]int, m)  // 只需要一行
```

- 掌握初始化和边界处理的重要性
- 理解如何利用约束条件简化问题

## 扩展问题

1. **计数**：有多少种最短路径？
2. **四向移动**：允许向上、下、左、右移动
3. **权重**：每个格子有不同的通过成本
4. **路径记录**：返回具体的路径而不仅是长度
5. **多个终点**：找到最近的终点
6. **动态障碍物**：随时间变化的障碍物

## 优化技巧

1. 只需保留一行的dp值（滚动数组）
2. 从终点反向搜索可能更快
3. 使用位压缩存储已访问状态
