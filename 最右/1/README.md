# 最右/1: 滑动窗口求最大连续人数

## 题目描述

一条商业街的AI摄像头每分钟会记录一次当前经过的人数，返回为一个非负整数数组 nums，nums[i] 表示第i分钟的人数。

需要统计在任意连续k分钟内总经过人数的最大值，以判断是否触发人流预警。

**问题定义**：
- nums 是一个长度为 n 的数组，每个元素表示某一分钟的人数
- k 是时间窗口大小（分钟数）
- 求所有连续k分钟内人数总和的最大值

**示例**：
```
输入: nums = [2, 1, 5, 1, 2, 3, 4, 1, 2, 1], k = 3
输出: 9
解释: 
分钟0-2: 2+1+5 = 8
分钟1-3: 1+5+1 = 7
分钟2-4: 5+1+2 = 8
分钟3-5: 1+2+3 = 6
分钟4-6: 2+3+4 = 9 (最大)
分钟5-7: 3+4+1 = 8
分钟6-8: 4+1+2 = 7
分钟7-9: 1+2+1 = 4
```

## 解题思路

### 核心思想
使用滑动窗口技术，通过维护一个固定大小的窗口，在单次遍历中计算所有窗口的和。

### 算法步骤

1. **初始化首个窗口**：
   ```
   windowSum = sum(nums[0:k])
   maxSum = windowSum
   ```
   计算前k个元素的和

2. **滑动窗口**：
   对于每个新位置 i（从k开始到n-1）：
   - 移除左边界元素：`windowSum -= nums[i-k]`
   - 添加右边界元素：`windowSum += nums[i]`
   - 更新最大值：`maxSum = max(maxSum, windowSum)`

3. **返回最大值**

### 时间复杂度
- **时间**: $O(n)$ - 单次遍历数组，每个元素只被访问常数次
- **空间**: $O(1)$ - 仅使用常数级额外空间

### 与暴力方法的对比

**暴力方法**：
```
for i = 0 to n-k:
    sum = 0
    for j = i to i+k-1:
        sum += nums[j]
    maxSum = max(maxSum, sum)
```
时间复杂度：$O((n-k) \times k)$ ≈ $O(n \times k)$

**优化方法**：
```
windowSum = sum(nums[0:k])  // O(k)
for i = k to n-1:
    windowSum = windowSum - nums[i-k] + nums[i]  // O(1)
    maxSum = max(maxSum, windowSum)
```
时间复杂度：$O(n)$

当k较大时，优化效果显著。

## 代码实现

```go
func maxConsecutiveSum(nums []int, k int) int {
    if len(nums) < k || k <= 0 {
        return 0
    }
    
    // 计算初始窗口和
    windowSum := 0
    for i := 0; i < k; i++ {
        windowSum += nums[i]
    }
    
    maxSum := windowSum
    
    // 滑动窗口
    for i := k; i < len(nums); i++ {
        // 移除左边界，添加右边界
        windowSum = windowSum - nums[i-k] + nums[i]
        
        // 更新最大值
        if windowSum > maxSum {
            maxSum = windowSum
        }
    }
    
    return maxSum
}
```

## 重要优化点

### 关键性质
- 窗口大小固定 → 可以使用滑动窗口
- 元素非负 → 不需要特殊处理负数情况
- 求最大/最小和 → 典型的滑动窗口应用

### 为什么高效
1. **重复计算消除**：不重复计算已计算过的元素
2. **增量更新**：只更新新增和删除的元素
3. **单次遍历**：整个过程仅遍历数组一次
