# Xiaomi-1: 一元一次方程求解

## 题目描述

给定一个形如 `ax + b = cx + d` 的一元一次方程，其中 x 是未知数。题目要求求解该方程，输出 `x=` 的值。

**方程格式**：
- 等号左右两边由 `=` 分隔
- 每边由多个项组成，项可以是：
  - 系数 * 变量（如 `2x`, `x`）
  - 常数（如 `3`, `-5`）
- 系数可以是正数或负数
- 变量统一为小写字母

**示例**：
```
输入: "2x+3=x+5"
输出: x=2.000
```

## 解题思路

### 核心思想
将方程两边进行分离，将含变量的项移到左边，常数项移到右边。

### 算法步骤

1. **分离方程**：
   - 使用 `=` 分隔符将方程分为左右两部分

2. **识别变量**：
   - 遍历左右两部分，找到所有包含的小写字母（变量）

3. **解析每一部分**：
   - 对每一部分调用 `parsePart()` 函数
   - 该函数返回变量系数和常数项

4. **合并同类项**：
   - 计算左边变量系数 - 右边变量系数 = 最终变量系数
   - 计算右边常数项 - 左边常数项 = 最终常数项

5. **求解**：
   - 如果最终变量系数不为0：`x = 常数 / 系数`
   - 如果系数为0：方程无解或有无穷多解

### 关键实现: parsePart() 函数

该函数用于解析方程的一部分：

```
算法:
1. 确保部分以符号开头（+ 或 -）
2. 从前往后遍历字符串
3. 对每个 ± 符号后的数字序列进行处理
4. 如果后面跟着变量，累加到系数；否则累加到常数项
```

**处理流程示例**（以 `2x-3` 为例）：
- 转换为 `+2x-3`
- 遍历：`+2x` → coeff += 2, `-3` → const += -3

### 时间复杂度
- **时间**: $O(n)$ - n是方程字符串长度
- **空间**: $O(1)$ - 仅使用常数级额外空间

## 代码实现

关键部分：
```go
// 提取符号
sign := 1.0
if part[i] == '-' {
    sign = -1.0
}

// 提取数字
numStr := ""
for i < n && unicode.IsDigit(rune(part[i])) {
    numStr += string(part[i])
    i++
}

// 判断是系数还是常数
if i < n && rune(part[i]) == r {
    coeff += sign * num  // 这是系数
} else {
    constVal += sign * num  // 这是常数
}
```

## 关键要点

1. **符号处理**：确保首位有符号便于统一处理
2. **字符分类**：使用 `unicode.IsDigit()` 识别数字
3. **默认系数**：无数字的变量表示系数为1
4. **浮点精度**：输出结果保留3位小数
