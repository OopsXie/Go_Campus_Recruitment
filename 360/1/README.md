# 360/1: 字符串变换可行性判断

## 题目描述

给定两个字符串 s1 和 s2，两个字符串长度分别为 n 和 m。

判断是否可以通过以下方式将 s1 变换为 s2：
- 从 s1 中删除一个长度为 k 的子串
- 在 s2 的末尾添加这个被删除的子串
- 使得删除后的 s1 加上新添加的子串等于 s2

**约束条件**：
- $n - m = 2k$（s1 删除k个字符后和 s2 长度相同后，再添加k个字符）
- 也就是说 $k = (n - m) / 2$

**示例**：
```
s1 = "abcde", s2 = "cdeab"
n = 5, m = 5 → n - m = 0，所以 k = 0
不能通过删除和添加实现

s1 = "abcde", s2 = "eab"
n = 5, m = 3 → n - m = 2，所以 k = 1
从 s1 中删除 "bc"，s1 变为 "ade"
但 s2 是 "eab"，不能变换

s1 = "abcde", s2 = "eabcd"
删除 "cd"，s1 变为 "abe"
s2 = "eabcd"，不相等
```

## 解题思路

### 核心思想
遍历所有可能的删除位置，对每个位置检查是否能通过删除该子串并将其添加到末尾得到 s2。

### 算法步骤

1. **验证长度条件**：
   - 计算 $k = (n - m) / 2$
   - 如果不能整除2或 $n < m$，返回false

2. **枚举删除位置**：
   对于每个可能的起始位置 i (从0到n-k)：
   - 提取删除的子串：`sub = s1[i:i+k]`
   - 构造删除后的字符串：`s1_part = s1[0:i] + s1[i+k:n]`
   - 构造目标字符串：`s2_part = s2 + sub`
   - 检查是否相等：`if s1_part == s2_part: return true`

3. **返回结果**

### 时间复杂度
- **时间**: $O(n^2)$ - 外层循环O(n)，内层字符串操作O(n)
- **空间**: $O(n)$ - 用于存储临时字符串

## 代码实现

```go
func isSubseq(s1, s2 string) bool {
    n, m := len(s1), len(s2)
    
    // 检查长度条件
    if (n-m)%2 != 0 || n < m {
        return false
    }
    
    k := (n - m) / 2
    
    // 尝试所有可能的删除位置
    for i := 0; i+k <= n; i++ {
        // 提取要删除的子串
        sub := s1[i : i+k]
        
        // 构造删除后的s1和目标s2
        s1part := s1[:i] + s1[i+k:]
        s2part := s2 + sub
        
        // 检查是否匹配
        if s1part == s2part {
            return true
        }
    }
    
    return false
}
```

## 例子演示

**例子 1**：
```
s1 = "abdacd", s2 = "dacd"
n = 6, m = 4, k = (6-4)/2 = 1

尝试删除位置:
i=0: sub="a", s1part="bdacd", s2part="dacda" → 不相等
i=1: sub="b", s1part="adacd", s2part="dacdb" → 不相等
i=2: sub="d", s1part="ababd", s2part="dacdd" → 不相等
i=3: sub="a", s1part="abdcd", s2part="dacda" → 不相等
i=4: sub="c", s1part="abdad", s2part="dacdc" → 不相等
i=5: sub="d", s1part="abdac", s2part="dacdd" → 不相等

答案: NO
```

## 关键要点

1. **长度约束**：$(n - m)$ 必须是偶数，且 $n \geq m$
2. **枚举范围**：i从0到n-k，即最多n-k+1次尝试
3. **字符串操作**：Go中的字符串拼接和切片操作

- 理解如何遍历所有可能的操作序列

## 优化思路

1. **哈希或指纹**：用哈希值代替字符串比较
2. **提前检查**：验证删除的子串是否确实在s1中
3. **KMP算法**：用于更高效的字符串匹配
