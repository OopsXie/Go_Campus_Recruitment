# JD-2: 网格最短路径问题（Direction Changes Required）

## 题目描述

在一个 n×m 的网格中，需要从起点 (sx, sy) 移动到目标点 (tx, ty)。

**问题的特殊性**：
- 每一步只能沿着某个方向（水平或竖直）连续移动
- 改变方向时需要付出代价（步数+1）
- `.` 表示可通行的道路，`#` 表示障碍物
- 求从起点到目标点所需的最少步数

**示例**：
```
起点到目标点需要改变方向的最少次数
```

## 解题思路

### 核心思想
使用BFS（广度优先搜索）进行状态搜索。状态包括：位置 (x, y) 和当前移动方向。

### 算法设计

1. **状态定义**：
   ```go
   State {x, y, dir, steps}
   - (x, y): 当前位置
   - dir: 当前方向（0=水平, 1=竖直）
   - steps: 已用步数
   ```

2. **初始状态**：
   - 从起点出发，可以先沿水平方向移动 (dir=0)
   - 或先沿竖直方向移动 (dir=1)

3. **转移规则**：
   - 如果当前方向是水平(dir=0)，可以继续向左或右移动
   - 移动到新位置时，方向变为竖直(dir=1)
   - 如果当前方向是竖直(dir=1)，可以继续向上或下移动
   - 移动到新位置时，方向变为水平(dir=0)

4. **访问记录**：
   `visited[x][y][dir]` - 防止重复搜索相同的状态

### 时间复杂度
- **时间**: $O(n \times m \times 2)$ = $O(nm)$ - 每个位置最多访问2次（2个方向）
- **空间**: $O(n \times m)$ - 访问记录和队列空间

## 关键要点

1. **方向转换**：每移动一步都会改变方向
2. **状态设计**：位置+方向的三维状态空间
3. **BFS适用性**：等权重的最短路径问题使用BFS
4. **访问优化**：记录访问状态而非仅位置
